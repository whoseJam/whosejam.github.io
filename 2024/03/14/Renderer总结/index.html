Renderer总结

2024-03-14 计算机图形学 评论

Renderer是我学计算机图形学的时候的动手实践，在此做一点简单的记录，以防自己以后看不懂自己的代码

Object & Component

类似Unity的设计，一个场景中可以有很多Object，每个Object可以通过挂不同的Component，被赋予不同的功能。Component类似于插件，Object和Component共同构成了一棵树，都可以有自己的parent，但Component一定是树的叶子节点

对每个Object都提供了showHierarchyUI的方法，用于显示其子树内Object的UI面板，例如\(O_2\)的showHierarchyUI会递归调用并显示以下这些Object的UI面板：

对每个Object都提供了showComponentUI，用于显示直接属于该Object的Component的面板，例如\(O_4\)的showComponentUI会调用并显示以下这些Component的UI面板（本质上是调用每个Component提供的showUI）：

资源

Shader

一个shader由一个顶点着色器和一个片段着色器组成，在shader内部提供了多种对顶点着色器和片段着色器的uniform变量赋值的代码，接口界面如下：

class Shader {
public:
    Shader(const char* vertexPath, const char* fragmentPath);
    void use();
    void setBool(const std::string& name, bool value) const;
    void setInt(const std::string& name, int value) const;
    void setFloat(const std::string& name, float  value) const;
    void setMat4f(const std::string& name, const Eigen::Matrix4f& value) const;
    void setVec3f(const std::string& name, float x, float y, float z) const;
    void setVec3f(const std::string& name, const Eigen::Vector3f& value) const;
    void setTexture(const std::string& name, TextureID texture, int rank) const;
    void setColor(const std::string& name, Color value) const;
private:
    ShaderID ID;
};

创建着色器

GLuint glCreateShader(GLenum shaderType);
shaderType = 
GL_VERTEX_SHADER: 顶点着色器
GL_FRAGMENT_SHADER: 片段着色器
GL_GEOMETRY_SHADER: 几何着色器
GL_TESS_CONTROL_SHADER: Tessellation控制着色器
GL_TESS_EVALUATION_SHADER: Tessellation评估着色器
GL_COMPUTE_SHADER: 计算着色器

设置着色器代码

void glShaderSource(GLuint shader, GLsizei count, const GLchar **string, const GLint *length);

编译着色器代码

void glCompileShader(GLuint shader);

把着色器搭建成管线

GLuint glCreateProgram(void);
void glAttachShader(GLuint program, GLuint shader);
void glLinkProgram(GLuint program);

若干个着色器可以attach到一个program上，随后这个program就可以被使用，当着色器被attach到program上之后，删除着色器对象并不会影响已经链接到程序对象的着色器程序。程序对象在链接成功后已经获取了所需的着色器信息，并不再直接依赖于着色器对象的状态

void glDeleteShader(GLuint shader);

设置着色器参数

使用glUniform*系列函数，uniform变量是在着色器中声明的全局变量，用于在应用程序和着色器之间进行数据传递

GLint glGetUniformLocation(GLuint program, const GLchar *name);
void glUniform1i(GLint location, GLint value);
void glUniform1f(GLint location, GLfloat value);
void glUniformMatrix4fv(GLint location, GLsizei count, GLboolean transpose, const GLfloat *value);

Mesh

一个mesh由若干vertices组成，一个vertex应该具有位置信息\(\vec{p}\)，法向量信息\(\vec{n}\)，材质位置信息\(\vec{t}\)

VertexArray

VertexArray是用来存储和管理顶点数据的机制，它是一种数据结构，用来组织顶点数据，以供渲染管线使用

VertexArray将顶点数据存储在显存中，并以一种更高效的方式传输给GPU，它提供一种将多个顶点数据打包成连续内存块的大方法，并通过指定偏移量和步长来描述每个顶点属性的布局

首先把VertexArray设置成活跃状态

void glBindVertexArray(GLuint array);

然后对VBO，EBO等等buffer进行数据绑定（见buffer）

随后，配置VertexArray的布局

void glVertexAttribPointer(GLuint index, GLint size, GLenum type, GLboolean normalized, GLsizei stride, const void* pointer);

如此这般，我们就完成了一个VertexArray的初始化，在未来渲染的时候，就可以利用VAO+shader，在GPU上做出我们希望得到的效果了

Buffer

Buffer是一段连续的内存区域，用来存储各种类型的数据，如顶点数据，纹理数据，索引数据等

Buffer主要用于把数据从CPU搬到GPU，通过Buffer，我们可以把数据一次性传输到GPU，而不需要逐个发送，这种批量传输提高了渲染效率

常见缓冲区类型 描述
Vertex Buffer Objects 用于存储顶点数据，如顶点位置，颜色，法线等
Index Buffer Objects 用于存储索引数据，绘制索引化几何图形
Uniform Buffer Objects 用于存储uniform数据，如变换矩阵
Shader Storage Buffer Objects 用于存储可读写的数据，供着色器使用
Texture Buffer Objects 用于存储纹理数据，可以在着色器中进行采样操作 
void glGenBuffers(GLsizei n, GLuint *buffers);

当生成一个buffer之后，需要将这个buffer和某种类型的数据关联上

绑定类型 描述
GL_ARRAY_BUFFER 用于顶点数据
GL_ELEMENT_ARRAY_BUFFER 用于索引数据
GL_UNIFORM_BUFFER 用于unifom数据
GL_SHADER_STORAGE_BUFFER 用于着色器存储数据 
void glBindBuffer(GLenum target, GLuint buffer);
void glBufferData(GLenum target, GLsizeiptr size, const void *data, GLenum usage);
usage = 
GL_STATIC_DRAW: 数据不会频繁变动，适用于静态数据
GL_DYNAMIC_DRAW: 数据会频繁更新，适用于动态数据
GL_STREAM_DRAW: 数据每帧都会变动，适用于流式数据

光照模型

光照模型永远不可能完全真实，我们只能尽可能保留更多的真实分量，或者拟造一些光，来近似在有限时间内，无法精确计算的，但存在于现实世界中的部分光

Phong光照模型

真实分量=环境光+漫反射光+镜面反射光

  • 环境光（Ambient Light）：场景中无论光源位置如何，都存在的背景光照

    • \[ Ambient\ Light=C\cdot [全局环境光强度] \]

    • \(C\)是物体表面的颜色，全局环境光强度是一个\([0,1]\)的值

  • 漫反射光（Diffuse Light）：光源射向物体表面，在各个方向上，均匀散射开来的光线

  • 镜面光（Specular Light）：光源射向物体表面，在特定方向上反射的光线
